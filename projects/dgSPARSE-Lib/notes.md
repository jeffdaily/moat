# dgSPARSE-Lib notes

## Build

Build on ROCm:
```bash
cd projects/dgSPARSE-Lib/src
pip install -e . --no-build-isolation
```

Hipified files (include/hip/, src/hip/, *_hip.cpp) are auto-generated by torch.utils.hipify during build. They are gitignored and will be regenerated on each build.

## Port summary

- **Strategy**: B (torch-hipify). CUDAExtension built against ROCm PyTorch auto-runs hipify.
- **Key changes**:
  - 64-bit FULLMASK required for ROCm 7.2.1+ warp sync functions
  - Guard CUDA-only headers (device_atomic_functions.h, device_launch_parameters.h)
  - Define CUBLAS_COMPUTE_16F and CUBLAS_TENSOR_OP_MATH for ROCm (hipify lacks mappings)
  - Link -lhipsparse instead of -lcusparse
- **Warp shuffles**: Width-32 shuffles used throughout are arch-agnostic (work on wave64 and wave32)

## Test dependencies

Tests require torch_geometric, ogb, and torch_sparse. torch_sparse needs to be built from source for ROCm which has compatibility issues with ROCm 7.2.1. The core library compiles and loads successfully.
