# dgSPARSE-Lib notes

## Build

Build on ROCm:
```bash
cd projects/dgSPARSE-Lib/src
pip install -e . --no-build-isolation
```

Hipified files (include/hip/, src/hip/, *_hip.cpp) are auto-generated by torch.utils.hipify during build. They are gitignored and will be regenerated on each build.

## Port summary

- **Strategy**: B (torch-hipify). CUDAExtension built against ROCm PyTorch auto-runs hipify.
- **Key changes**:
  - 64-bit FULLMASK required for ROCm 7.2.1+ warp sync functions
  - Guard CUDA-only headers (device_atomic_functions.h, device_launch_parameters.h)
  - Define CUBLAS_COMPUTE_16F and CUBLAS_TENSOR_OP_MATH for ROCm (hipify lacks mappings)
  - Link -lhipsparse instead of -lcusparse
- **Warp shuffles**: Width-32 shuffles used throughout are arch-agnostic (work on wave64 and wave32)

## Test dependencies

Tests require torch_geometric, ogb, and torch_sparse. torch_sparse needs to be built from source for ROCm which has compatibility issues with ROCm 7.2.1. The core library compiles and loads successfully.

## Validation 2026-06-08 (windows-gfx1201)

### Build

Fork commit 81b3404 adds Windows+ROCm build support on top of the Linux port (c262d8b).

Four Windows-specific fixes required:
1. `setup.py`: `HIPBuildExtension` subclass registers `.hip` as MSVC C++ extension; `use_ninja=True` so `win_wrap_ninja_compile` escapes spaces in MSVC include paths for hipcc; `_winhip.cu` shims route host `.cpp` files through hipcc (MSVC cl.exe rejects GCC `__attribute__` syntax in HIP headers); `hipsparse.lib` instead of `-lhipsparse`; POSIX-only `-s/-lm/-ldl` flags omitted.
2. `include/cuda/csr2csc.cuh`: torch hipify lacks mappings for `cusparseCsr2cscEx2`, `CUSPARSE_ACTION_NUMERIC`, `CUSPARSE_CSR2CSC_ALG1`; added `USE_ROCM` compat `#define` aliases.
3. `src/spmm.cpp`: added minimal `PYBIND11_MODULE` stub under `_WIN32`; MSVC requires `PyInit__spmm_cuda` resolvable at link time even though the extension is loaded via `torch.ops.load_library`.
4. `.gitignore`: added `*_winhip.cu`, `*_winhip.hip`, `*.pyd`.
